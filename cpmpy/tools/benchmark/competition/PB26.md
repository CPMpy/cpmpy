# CPMpy @ Pseudo Boolean Competition 2026

This document contains the installation and usage instructions for the CPMpy submission to the PB 2026 competition.

## Submission

This submission is the basis for multiple submissions with different solver backends. CPMpy is a modelling system which can translate to many different solvers, three of which have been chosen for the PB26 competition. The data files and install instructions are shared. From the executable's point of view, the major difference between the submissions is the actual command used to run the executable, where the correct solver must be set. These commands are listed later on. Internally, different solver-tailored backends will be used, where the COP and CSP models get transformed as to satisfy the modelling capabilities of the selected solver target.

The CPMpy modelling system will compete in the following tracks, using the following solver backends:

| **cpmpy_backend** | DEC-LIN | OPT-LIN |
| - | - | - |
| **cpmpy_ortools** | yes | yes |
| **cpmpy_highs** | yes | yes |
| **cpmpy_pindakaas_cadical** | yes | **no** |


## Setup

### Dependencies

CPMpy is a Python library, and thus Python (>=3.10) is the main dependency.

### Installation

1. Create a Python virtual environment:

```console
python -m venv .venv
```

2. Activate Python environment

Everything should be installed in the Python virtual environment. Either activate the environment using:

```console
source .venv/bin/activate
```

or replace `python` with the path to the Python executable (./.venv/bin/python) and `pip`
 with `<path_to_python> -m pip` in each of the following steps.

3. Navigate to the root directory of this submission.

4. Install Python libraries

```console
pip install .[highs,pindakaas,io.opb]
pip install tqdm psutil filelock 
```

Now we should be all set up!


## Usage

The interface of the executable is as follows:

```console
python run_benchmark.py BENCHNAME
    --runner opb 
    [--seed RANDOMSEED] 
    [--time_limit TIMELIMIT] 
    [--mem_limit MEMLIMIT] 
    --verbose                   # To print output to stdout
    [--cores NBCORES]       
    [--intermediate]
    [--solver SOLVER]           # Name of the CPMpy solver backend
```

The same executable supports multiple solver backends and is used for all of the submissions to the competition. The submitted cpmpy + backends are:

- `cpmpy_ortools`
- `cpmpy_highs`
- `cpmpy_pindakaas_cadical`

The commands are as follows:

| **Submission** | **Command** |
| - | - |
| **cpmpy_ortools** | python ./cpmpy/tools/benchmark/cpbenchy/runner/run_benchmark.py BENCHNAME --runner opb --time_limit TIMELIMIT --mem_limit MEMLIMIT --verbose --cores NBCORES --seed RANDOMSEED --intermediate --solver ortools |
| **cpmpy_highs** |python ./cpmpy/tools/benchmark/cpbenchy/runner/run_benchmark.py BENCHNAME --runner opb --time_limit TIMELIMIT --mem_limit MEMLIMIT --verbose --cores NBCORES --seed RANDOMSEED --solver highs |
| **cpmpy_pindakaas_cadical** | python ./cpmpy/tools/benchmark/cpbenchy/runner/run_benchmark.py BENCHNAME --runner opb --time_limit TIMELIMIT --mem_limit MEMLIMIT --verbose --cores NBCORES --seed RANDOMSEED --solver pindakaas |
